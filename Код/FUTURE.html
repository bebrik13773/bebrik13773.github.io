Промты Для Генерации SVG-Кода

Это отдельный блок именно для нейросети, которая должна вернуть не картинку, а готовый SVG-код. Логика такая: сначала копируется общий мастер-промт, затем общий технический блок, затем нужный промт состояния или ассета.

0. Общий мастер-промт для SVG-кода

Generate production-ready raw SVG code for a 2D side-view casual mobile game asset. Return only valid SVG markup, no markdown fences, no explanations, no JSON, no commentary. The result must be a single self-contained <svg> element with width, height and viewBox. Use clean layered vector shapes, grouped structure, readable ids for major parts, and compact organization suitable for manual editing later. The art style must be polished cartoon mobile game art, readable at small size, with a clear silhouette and friendly whimsical swamp-forest vibe. The asset must look unmistakably like part of the same game world and same artist style.

1. Общий технический блок для всех SVG-ассетов

Technical requirements for the SVG:
- output only raw SVG code
- transparent background
- side view only
- full subject visible with safe padding
- centered composition
- mobile-game readability first
- clean silhouette, not noisy
- no raster image embedding
- no external fonts
- no external CSS
- no scripts
- use vector paths, ellipses, circles, gradients and simple masks only when helpful
- avoid huge filter stacks and heavy blur that would be hard to render in-browser
- keep shapes editable and layered
- use readable group ids like body, tail, ear, eye, paw, outline, highlight
- add subtle gradients and highlights, but keep the result lightweight
- no text, no watermark, no frame, no UI elements
- no background scene unless explicitly asked

2. Общий negative prompt для SVG-кода

Do not make it photorealistic, painterly noise-heavy, furry with thousands of strands, seal-like, rat-like, flat clipart, front-facing, top-down, isometric, over-detailed, asymmetrical in a broken way, cropped, too dark, muddy, horror-like, dirty, messy, with extra limbs, malformed anatomy, unreadable tail, unreadable paws, unreadable teeth, giant head with tiny tail, random props, text, labels, frame, white background, bitmap texture, jpeg artifacts, or empty placeholder geometry.

Общие требования ко всем ассетам

  • Стиль: яркий, читаемый, мультяшный 2D game art, friendly casual mobile game, не реализм, не 3D render, не low-poly, не пиксель-арт.
  • Камера: только боковой вид side view, без сильной перспективы, без изометрии.
  • Читаемость: силуэт должен быть понятен даже в маленьком размере на телефоне.
  • Контур: аккуратный, чистый, без грязных рваных краёв, без лишних мелких деталей, которые распадутся при уменьшении.
  • Фон: если это отдельный спрайт, то фон обязательно прозрачный.
  • Композиция: объект по центру, не обрезан, с запасом воздуха по краям.
  • Свет: мягкий, сказочный, слегка влажный лесо-болотный вайб, но без мрачного хоррора.
  • Палитра: болотные зелёные, тёплые коричневые, мягкие бирюзовые и голубые акценты, чуть золотистого света.
  • Формат результата: ideally PNG или SVG-ready look, квадратный холст 2048x2048 для одиночных объектов, прозрачный фон.
  • Важно: без текста, без интерфейсных надписей, без водяных знаков, без рамок, без лишних персонажей.
